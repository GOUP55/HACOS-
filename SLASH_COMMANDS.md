# スラッシュコマンド早見表

チャット欄に `/コマンド名` と打つだけで、決まった手順を正しく実行できます。
定義ファイルは `.claude/commands/` にあります（編集すればカスタマイズ可能）。

## トップ10

| コマンド | 何をするか | 使いどき |
|---|---|---|
| `/heavy-task 〇〇して` | チーム総出の品質ループ（制作→build→QA→修正）で大きめの仕事を完遂 | 「プレミアムLPの訴求を全面見直しして」など重い依頼 |
| `/status` | 公開済み・作業中・次にやることを非エンジニア向けに報告 | セッションの最初、状況が分からなくなったとき |
| `/qa` | 公開前チェック（リンク・崩れ・タイポ・数値整合）。指摘リストだけ出す | 公開・共有の前に必ず |
| `/fact-check` | 料金・締切・定員が BUSINESS_RULES.md と一致しているか全ファイル突合 | 価格や締切を変えた後、公開前 |
| `/build` | メインLPをビルドして index.html を更新・整合確認 | hacos-hmc-lp.html を変えた後 |
| `/lp-fix 〇〇を直して` | LPの修正・レイアウト調整・セクション追加を実装 | 「ここの余白おかしい」「セクション足して」 |
| `/copy 〇〇の文章` | LP・LINE・IG向けコピーを2〜3案作る（反映は選んでから） | 見出し・CTA・特典文言が欲しいとき |
| `/line 〇〇の告知` | LINE配信文を2案作る（送信はしない） | 体験会告知・リマインド・ローンチ配信 |
| `/ig 〇〇のテーマ` | Instagram投稿案（キャプション＋構成）を作る | 投稿ネタが欲しいとき。テーマ省略でカレンダーから提案 |
| `/handoff` | 引き継ぎメモ作成・連絡板更新・コミット | セッションを終える前に毎回 |

## 11〜20（反映・公開・改善・トラブル対応）

| コマンド | 何をするか | 使いどき |
|---|---|---|
| `/price-change 〇〇を△△に` | 料金・締切変更を BUSINESS_RULES.md 起点で全ファイル一括反映 | 値段や募集締切を変えるとき（必ずこれ経由で） |
| `/preview` | LPをモバイル幅390pxで開いてスクショを見せる | 変更後に見た目を自分の目で確認したいとき |
| `/publish` | mainへ公開する内容を整理→あなたの承認→PR作成 | 公開したいとき（勝手にmainには反映しない） |
| `/undo 〇〇を戻して` | 直近の変更を安全に取り消す（確認してから実行） | 「さっきの変更やっぱりナシ」のとき |
| `/kaizen 〇〇が伸びない` | 集客・導線の改善案を効果×手間の表で提案（実装なし） | LINE登録・予約が増えないと感じたとき |
| `/banner 〇〇の画像` | OGP・バナー・実績画像をブランド配色で制作 | SNS用・告知用の画像が欲しいとき |
| `/faq 〇〇って聞かれた` | 質問への回答文を作成、LPのFAQ追加も提案 | お客様から質問が来たとき |
| `/weekly` | 今週の作業・予定とのズレ・来週やることを1枚に | 週1回、週初めか週末に |
| `/explain 〇〇って何？` | ファイル・変更・専門用語を平易な言葉で解説 | 分からない言葉・ファイルが出てきたとき |
| `/reserve-check` | LINE予約システムの設定・状態を点検（変更はしない） | 予約まわりが不安なとき、料金変更後の確認に |

## おすすめの流れ

- **セッション開始**: `/status` → 現在地を把握してから指示
- **何かを変えたら**: 変更 → `/build` → `/preview` で目視 → `/qa` → 指摘ゼロになったら `/publish`
- **料金・締切を変えるとき**: `/price-change` → `/fact-check` で残りゼロ確認
- **週1回**: `/weekly` で予定とのズレを確認
- **セッション終了**: `/handoff` → 次回は表示された指示文をそのまま貼る

## 注意

- どのコマンドも **main への公開や外部送信は自動ではしません**（あなたの承認が必要）
- 料金・締切などの正本は `BUSINESS_RULES.md`。変更したらまずそこを直してから `/fact-check`
