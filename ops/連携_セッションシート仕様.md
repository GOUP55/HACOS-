## HACOSトレーニングメソッド（フェーズ定義）

| 略称 | 正式名称 | 内容 |
|------|----------|------|
| **AL** | リアアライメント | 姿勢・後面連鎖の調整 |
| **CA** | コアアクティベート | コア活性化 |
| **IG** | インテグレーション | 動作統合 |
| **STR** | ストレングス | 筋力トレーニング（種目グリッド） |
| **PA** | パワー | パワー系トレーニング |

> ⚠️ 旧シートに "PWR" と記載があるが、正式名称は **PA（パワー）**。実装は必ず `pa` で統一すること。

## データモデル（Firestore コレクション構造）

### `clients` コレクション

```typescript
interface Client {
  id: string;              // Firestore doc ID（自動生成）
  name: string;            // 氏名（必須）
  birthDate: string | null;  // 生年月日 "YYYY-MM-DD"
  height: number | null;   // 身長 cm
  course: string | null;   // コース名（例: "ダイエット3ヶ月"）
  createdAt: Timestamp;
}
```

### `sessions` コレクション

```typescript
interface Session {
  id: string;
  clientId: string;         // clients のドキュメントID
  date: string;             // "YYYY-MM-DD"（必須）
  sessionNumber: string | null;  // セッション番号（例: "5"）
  trainer: string | null;   // 担当トレーナー名

  // 体組成（InBody等の測定値）
  bodyWeight: number | null;      // 体重 kg
  bodyFat: number | null;         // 体脂肪率 %
  visceralFat: number | null;     // 内臓脂肪レベル
  skeletalMuscle: number | null;  // 骨格筋量 kg

  // フェーズ評価（自由テキスト or スコア）
  al: string | null;   // リアアライメント
  ca: string | null;   // コアアクティベート
  ig: string | null;   // インテグレーション
  pa: string | null;   // パワー

  // コミュニケーション
  communicationNotes: string | null;  // 話した内容・伝達事項
  nextSuggestion: string | null;      // 次回までの提案

  // STR トレーニング（最大6種目）
  exercises: Exercise[];

  createdAt: Timestamp;
}

interface Exercise {
  name: string;          // 種目名（例: "スクワット"）
  weight: number | null; // 重量 kg
  reps: number | null;   // 回数 rep
  sets: number | null;   // セット数
}
```

### `measurements` コレクション（2週間変化記録）

```typescript
interface Measurement {
  id: string;
  clientId: string;
  date: string;               // "YYYY-MM-DD"

  // 周径測定 cm
  waist: number | null;       // ウエスト
  hip: number | null;         // ヒップ
  kneeAbove15: number | null; // 膝上15センチ
  upperArm: number | null;    // 二の腕

  dietNotes: string | null;   // 2週間の変化と食事内容

  createdAt: Timestamp;
}
```

## 画面仕様（4画面）

### 1. クライアント一覧（`/` or `#/`）
- 全クライアントをカード表示
- 名前検索
- 新規クライアント追加ボタン

### 2. クライアント詳細（`/client/:id`）
- プロフィール（名前・生年月日・身長・コース）
- 体組成推移グラフ（体重・体脂肪・骨格筋 の折れ線）
- セッション履歴一覧（日付降順）
- 2週間変化記録一覧

### 3. セッション記録フォーム（`/client/:id/session/new` または編集）
- 基本情報（日付・セッション#・担当トレーナー）
- 体組成（体重・体脂肪・内脂肪・骨格筋）
- フェーズ評価（AL / CA / IG / PA）
- 話した内容・伝達事項
- STR トレーニング（種目①〜⑥ × 重量・回数・セット）
- 次回までの提案☆

### 4. 2週間変化記録フォーム（`/client/:id/measure`）
- 日付
- 周径測定（ウエスト・ヒップ・膝上15cm・二の腕）
- 2週間の変化と食事内容（自由テキスト）

## 運用ルール

- **ログイン不要**・共有URLで全トレーナーがアクセス
- **リアルタイム同期**（Firebase Firestore）
- トレーナー数: 3〜10名想定
- クライアント数: 20〜100名想定
- 主要デバイス: スマートフォン・PC
