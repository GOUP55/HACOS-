# HACOS チーム ＆ 品質ループ（ループエンジニアリング）

このリポジトリは「会社のような役割分担」で開発します。各役割は `.claude/agents/` に定義され、
Agentツールの `subagent_type` で呼び出せます。本書は**役割**と、最高品質に到達するための
**反復品質ループ（ループエンジニアリング）**の手順を定めます。

## 1. チーム（6体）

| エージェント | 役割 | 主担当 |
|---|---|---|
| `hacos-director`   | 司令塔(PM) | 要件整理→分解→割振→統合→品質ゲート |
| `hacos-copywriter` | コピー | LP/LINE/IGの文章・ストーリー・特典文言 |
| `hacos-frontend`   | 実装 | HTML/CSS/JS・build_index・画像最適化・表示確認 |
| `hacos-designer`   | ビジュアル | OGP・バナー・実績画像加工・配色/余白/レイアウト |
| `hacos-growth`     | 集客/計測 | LINE/IG導線・GA4/Pixel・CRO・オファー設計 |
| `hacos-qa`         | 品質保証 | 公開前チェック・指摘リスト（実装はしない） |

## 2. 品質ループ（Definition of Done に達するまで回す）

> 原則：**速さより品質**。QAが「重大度 高・中ゼロ」を出すまでループを止めない。

```
   0. director: 要件・成功条件・対象ファイルを定義
            │
            ▼
   1. 制作   copywriter / frontend / designer が実装
            │
            ▼
   2. build  frontend が `python3 build_index.py`（メインLP変更時）
            │
            ▼
   3. レビュー hacos-qa が指摘リスト(高/中/低)を出力
            │
      ┌─────┴─────┐
   高/中あり      高/中ゼロ
      │             │
      ▼             ▼
   4. 修正        5. director 最終確認 → 公開ゲート（ユーザー承認後に main へ）
   （担当へ差戻し）
      │
      └──────► 3 に戻る（再レビュー）
```

### ループの終了条件（Definition of Done）
- QA指摘の **重大度「高」「中」がゼロ**（「低」は許容 or 明示的に見送り合意）
- リンク・価格・日付・定員・実績数値・注記の整合が取れている
- モバイル(390px)で表示崩れ・コンソールエラーなし
- `hacos-hmc-lp.html` → `build_index.py` → `index.html` が一致
- 確認用スクラッチ画像が残っていない

### 公開ゲート（不可逆操作の前に必ず）
- `main` へのマージ／外部送信は **ユーザー承認後**のみ
- サンプル・下書き・プレースホルダ・ダミー計測IDを含む状態では公開しない

## 3. ループの回し方

### A. 単発で回す（推奨・通常運用）
director 起点で 1→2→3→（必要なら4→3）→5 を1サイクル。QAが「高/中ゼロ」を返すまで
4↔3 を繰り返す。各サイクルはブランチにコミットし、差分を残す。

### B. スケジュールで回す（任意・継続改善）
Claude Code の `/loop` を使うと、一定間隔で品質ループを自動起動できる。例：
```
/loop 30m hacos-qa で premium.html と index.html を公開前チェックし、
重大度 高/中 の指摘があれば該当ロールに修正を割り振って再チェックして
```
※ トークンを消費し続けるため、ローンチ直前など必要な期間だけ有効化する。

## 4. 不変のルール（全エージェント共通）
- `index.html` を直接編集しない（`hacos-hmc-lp.html` を編集→ビルド）
- 事実・数値・実績を捏造しない（実績は提供素材のみ・「効果には個人差があります／掲載許可」注記）
- 計測はダミーIDで発火させない（ID確定まではテンプレをコメントのまま）
- 開発は `claude/handoff-structural-improvements-0aynnf`。`main` は公開可能な確定物のみ
- 迷う分岐はユーザーに短く確認（勝手に決めない）
