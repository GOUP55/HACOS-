# 法人ウェルネス｜1週間ウェルネスジャーニー 福利厚生版（第1号プログラム）

> **来週実行**。福利厚生でHACOSを利用中の従業員（腰痛あり）1名を第1号として、
> 「1週間のウェルネスジャーニー（戦略5）× 健康弁当（戦略4）× 習慣トラッキング（戦略3）」を
> 法人ウェルネス（戦略1）のパイロットとして走らせる。
> チャット名：**「6時半だヨ！全員週5」**（平日5日・朝6:30起点）
>
> 作成: 2026-07-14／司令塔セッション。価格・費用負担は **BUSINESS_RULES.md 未記載＝オーナー決定待ち**。
> 本書は設計であり、運動メニューの最終確認はトレーナー（オーナー）が対面で行う。

## 0. 安全上の大前提（腰痛のある参加者のため・厳守）

- 本プログラムは**医療・治療ではなく、健康習慣づくりのサポート**。腰痛の改善を約束しない。
  文面でも「腰痛が治る」「痛みが消える」等の断定表現を使わない（効果には個人差があります）
- **痛みが出たら即中止**をすべての運動提案に添える。しびれ・強い痛みがある場合は
  医療機関への相談を優先してもらう一文を初日に必ず入れる
- 運動内容は「痛みのない範囲での軽い動き（歩く・ゆっくり立ち上がる・呼吸）」に限定。
  具体的なエクササイズ指導は、**トレーナーが本人の状態を対面で確認した上で**個別に出す
  （チャットの一斉文面には汎用の軽い動きだけを載せる）
- 記録される内容（チェックイン・メモ）がスタッフに共有されることを初日に明示する

## 1. プログラムの型（1週間・平日5日）

**1日の流れ（毎日同じリズム。朝→昼→夜の3接点）**

| 時刻 | 接点 | 内容 |
|---|---|---|
| 6:30 | チャット「6時半だヨ！全員週5」 | その日のテーマ配信＋起床チェックイン（スタンプ1個でOK） |
| 朝 | 2〜3分ルーティン | 痛みのない範囲の軽い動き（呼吸・ゆっくり伸び・その場歩き等）。「痛みが出たら中止」を毎回添える |
| 昼 | 健康弁当（HACOS KITCHEN） | 弁当＋「今日の食べ方ひとこと」（よく噛む・野菜から等の一言だけ） |
| 夜 | チェックイン | 🏃動いた／🥗食べ方に気をつけた をスタンプか一言で報告（10秒） |

**5日間のテーマ（設計）**

| 日 | テーマ | ねらい |
|---|---|---|
| 月 | スタート・今の自分を知る | 自己観察（今日の調子を10段階でひとこと）。無理しない宣言。安全上の注意を明示 |
| 火 | 座り方・立ち上がり | デスクワーク中の姿勢と「30〜60分に1回立つ」習慣。腰に負担の少ない立ち上がり方 |
| 水 | 歩く | 昼休みに5〜10分歩く。歩数ではなく「歩いたか」だけを問う |
| 木 | 食べ方 | 弁当を教材に、食べる順番・噛む回数・水分。数値や制限は扱わない |
| 金 | 振り返り・週末の過ごし方 | 5日間の変化をひとことで。週末に1つだけ続けることを自分で選ぶ→次週へ接続 |

**記録方法（二段構え）**
- **初週（来週）**: チャット内のスタンプ・一言返信で記録（手動）。スタッフが夕方に未報告者へやさしく一声
- **LIFF開通後**: `line-habit/` の習慣トラッキング（🏃🥗記録・連続日数・週間ダイジェスト・離脱アラート）に移行。
  ※LIFFはコード完成済みだが**本番未デプロイ**（LINE Developers手作業が必要。`line-habit/README.md`）。
  来週の実行はLIFFを待たない

## 3. 運用（AIチームで回る形＝戦略2）

**週次ループ（毎週これだけ）**

```
金曜: AIチームが翌週5日分の配信文面を生成（hacos-copywriter）
    → hacos-qa が表現チェック（医療的断定・誇大表現・安全注意の漏れ）
    → オーナー承認（ここだけ人間・5分）
週末: LINE予約配信 or スタッフへ文面共有
平日: 6:30配信は予約送信で自動。夜のチェックイン確認と一声はスタッフ or オーナー
月曜: 前週の参加状況を週次レビューに1行で報告
```

- 文面の正本はこのリポジトリの `drafts/corporate-wellness-week1-messages.md`（週ごとに追加）
- 参加者の体調・痛みに関する判断は**AIに任せない**。本人からの体調の訴えは必ずオーナーへ
- 2週目以降のテーマ改訂は、参加者の反応（チェックイン率・コメント）を見て司令塔が設計

**KPI（数値目標は置かず、観察のみ）**
- 朝チェックイン率／夜報告率（5日中何日）
- 弁当の継食数
- 開始時と終了時の本人ひとことアンケート（調子10段階＋自由記述）※医学的測定はしない

## 4. 8戦略との接続（このパイロットが生む資産）

| 戦略 | このプログラムから得るもの |
|---|---|
| 1 法人ウェルネス | 第1号事例。「導入企業の声」＋運用実測（工数・反応）→ 正式価格設計の根拠 |
| 2 AIチーム | 週次ループ（生成→QA→承認→配信）の型が完成。以後の全プログラムに流用 |
| 3 システム外販 | 習慣トラッキングの法人利用実績。「手動→LIFF移行」の導入ストーリーが営業資料になる |
| 4 弁当B2B | 職場への週5食提供の物流テスト（〆切・受け渡し・数量変動） |
| 5 ジャーニー | 福利厚生版テンプレv1。B2C版・リトリート版への展開元 |
| 6 ライセンス | 「1週間プログラムの運用マニュアル」がそのままライセンスパッケージの1章になる |
| 7 地域提携 | 導入企業が最初の提携先。社内紹介→他部署・取引先への横展開 |
| 8 発信メディア | 「AIチームで法人ウェルネスを1週間で立ち上げた」実録ネタ（本人・企業の掲載許可を取ること） |

> 掲載・発信に使う場合は、**本人と企業の掲載許可を書面（LINEでも可）で取ってから**。
> 実績表現には「効果には個人差があります」を付ける。
