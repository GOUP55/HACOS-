# Instagram運用AIエージェントシステム｜仕組みと使い方

HACOS × HMC のInstagram運用を、AIエージェントのパイプラインとして回すための仕組みです。
Claude（Fable 5）の特性——長文コンテキストの保持・マルチエージェントの分業・自己評価ループ・Web検索——を活かし、「リサーチ→企画→制作→評価→計測→改善→転用」を一気通貫で実行します。

---

## 全体像（パイプライン）

```
STEP0 リサーチ ── STEP1 企画5案 ── STEP2 フック&導入 ── STEP3 台本
   │                                                        │
   │                                          STEP4 評価・添削（0-5点）
   │                                                        │
   └── STEP5 実績分析（投稿後の数字） ←──────── 投稿 ──────────┘
                    │
              STEP6 ストーリーズ転用（4心理パターン）
```

**司令塔は `insta-director`。**「リールを1本作って」と言えば、必要な工程を順に回して統合納品します。

---

## エージェント一覧と呼び出し方

| エージェント | 役割 | こう頼む |
|-------------|------|---------|
| **insta-director** | 司令塔。全工程の段取り・統合・品質ゲート | 「朝活テーマでリールを1本、完成まで進めて」 |
| **insta-research** | 【STEP0・最重要】伸びている投稿の抽出。`site:www.instagram.com/reel/ キーワード after:日付` 検索で市場ニーズを特定 | 「50代ダイエットで伸びてるリールをリサーチして」 |
| **insta-ideas** | 【STEP1】リールアイデア5案（タイトル/構成/効果/難易度） | 「リールのネタを5つ出して」 |
| **insta-hook** | 【STEP2】冒頭フック（15文字前後×5型）＆導入（コールドリーディング4段構成） | 「朝活テーマでフックと導入を作って」 |
| **insta-script** | 【STEP3】eduGate式30秒台本＋キャプション＋セルフ改善TOP3 | 「台本を書いて」 |
| **insta-review** | 【STEP4】3軸0-5点採点・改善TOP3・書き直し案 | 「前回の台本を評価して」 |
| **insta-analyst** | 【STEP5】実績データ分析（強み/弱み/離脱推測/改善TOP3/再チャレンジ台本） | 「再生12,000・保存150・維持率35%だった。分析して」 |
| **insta-stories** | 【STEP6】台本→ストーリーズ超長文投稿（4心理パターン、文字数調整可） | 「この台本を1000文字でストーリーズに転用して」 |

### 共通の前提知識
- **`instagram/PERSONA.md`** … 50代主婦ペルソナ・訴求軸・遵守事項（薬機法/景表法/捏造禁止）。全エージェントが作業前に必ず読む
- 成果物は `instagram/{research,ideas,hooks,scripts,reviews,analytics,stories}/` に日付つきで蓄積され、**次回以降のエージェントが自動で参照**する（＝使うほど賢くなる仕組み）

---

## 運用ルーティン（週次の回し方）

1. **週初**：insta-research でリサーチ（キーワード3〜5パターン、直近1週間）
2. insta-ideas で5案 → 作る1本を決める
3. insta-hook → insta-script → insta-review（3点未満の軸があれば書き直し）→ 撮影・投稿
4. **投稿48〜72時間後**：インサイトの数字を insta-analyst に渡して分析
5. 伸びた台本は insta-stories でストーリーズに転用（1本のリールから最大4投稿を再生産）
6. 分析結果を踏まえて翌週のリサーチ・企画へ

### 依頼のコツ（時短テクニック）
- 初回は「詳細に」、2回目以降は「**前回のターゲット向けに**」で時短
- 「**動詞**」で具体的に頼む：「どう？」はNG →「5段階で評価して」「3案出して」
- 過去の成果物は「前回の台本を参照」で呼び出せる

## Fable 5の特性をどう活かしているか

| 特性 | この仕組みでの活用 |
|------|------------------|
| 長文コンテキスト | ペルソナ・過去のリサーチ・全台本履歴を踏まえた一貫性のある制作 |
| マルチエージェント分業 | 制作者と評価者を分離し、忖度のない採点ループを実現（STEP3⇄4） |
| 自己評価・改善ループ | 台本作成後に必ず「改善TOP3」を自己提案。評価3点未満は自動で書き直し |
| Web検索・リサーチ | 「勘」ではなく実際に伸びている投稿から型を抽出（STEP0） |
| ファイルシステム記憶 | `instagram/` 配下に成果物を蓄積し、回すほど精度が上がる |

## SNS以外への展開（ビジネス・マーケ全般）

この「リサーチ→企画→制作→評価→計測→改善→転用」の型はInstagram専用ではありません。

- **LINE配信**：伸びたリール台本 → hacos-copywriter でステップ配信文に転用（`LINE_MESSAGE_SEQUENCE.md` と連携）
- **LP改善**：insta-analyst で判明した「刺さる訴求」を hacos-growth / hacos-frontend でLP（`hacos-hmc-lp.html`）のファーストビューに反映
- **広告文**：insta-hook の5つのフック型は広告見出しにそのまま流用可能
- **商品開発**：insta-research の市場ニーズ仮説は、プレミアムプラン（`PREMIUM_LAUNCH_PLAN.md`）の訴求設計の一次情報になる
- **公開前チェック**：hacos-qa が薬機法/景表法・リンク・整合性を最終確認

Instagram → LINE登録 → カウンセリング → 成約、という既存導線（`PERSONAL_BRAND_PLAN.md` / `INSTAGRAM_CONTENT_CALENDAR.md`）の最上流を、このエージェント群が強化します。
