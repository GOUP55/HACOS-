## 📋 貼り付け用プロンプト（ここからコピー）

HACOSのLINE予約フォームに、7/6デプロイ後にmainへ入った品質修正を本番反映してください。
変更済みソースは GitHub `goup55/HACOS-` の main にあります（`line-reservation/` 配下）。
あなたはコードを書き換えず、mainのファイルをそのまま反映するだけです。

**今回反映する変更（PR #19 と PR #27 の一部）**
1. `liff/reserve.html`: 品質修正一式＋TACOS Party説明の料金表記統一（参加費¥3,000・タコス食べ放題・ドリンク/その他フード別）
2. `src/reservation-routes.js`: 品質修正一式＋月末リマインドの回数券計算から特別枠（TACOS等）を除外＋時刻ラベル対応
3. D1マイグレーション2本（**各1回だけ**実行。適用済みかは内容を見て判断）:
   - `wrangler d1 execute line-harness --file=line-reservation/migrations/2026-07-06-time-label.sql --remote`
   - `wrangler d1 execute line-harness --file=line-reservation/migrations/2026-07-07-tacos-pricing-note.sql --remote`

**手順（7/6と同じルート。`line-reservation/README.md` の訂正事項を厳守）**
1. mainから最新の `line-reservation/liff/reserve.html` を取得し、KVへ:
   `wrangler kv key put --binding=STATIC_KV "liff/reserve.html" --path=<取得したreserve.html> --remote`
2. mainの `line-reservation/src/reservation-routes.js` の内容で apps/worker 側の同ファイルを置換
3. 上記マイグレーション2本を実行
4. デプロイは **`apps\worker` で `pnpm run deploy`**。
   ⚠️ `npx wrangler deploy` 単体は使わない（ビルドが飛び古いコードが出る事故が実際に発生済み）

**動作確認（必ずLINEアプリ内のLIFFで。ブラウザは管理者Cookieで素通りするため確認にならない）**
- [ ] 予約フォームが開き、体験/会員/回数券/ビジター/相談の切替が今まで通り動く
- [ ] TACOS Partyの説明に「参加費¥3,000（タコス食べ放題）・ドリンク/その他フード別」が表示される
- [ ] セッションの時刻表示が正しく出る
- [ ] 回数券の「今月の開催日をまとめて選択」でTACOS等の特別枠が選ばれない

問題があれば、KVに旧reserve.htmlを戻し、routes置換前のコードで `pnpm run deploy` すれば元に戻ります。
