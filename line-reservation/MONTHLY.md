# 月次運用ガイド（毎月の日程追加のやり方）

予約フォームの日程はD1データベースの `sessions` テーブルで管理している。
**翌月の日程を入れないと、当月最後の開催日の正午以降、フォームは
「現在受付中の日程がありません」表示になる**ので、毎月20日頃までに翌月分を入れる。

> 📱 **2026-07-17から、単発の日程追加・編集・受付締切/再開・削除は
> 予約管理画面の「🛠 開催日の管理」からスマホでできるようになった**（SQL不要）。
> 1日ずつ入力すれば来月分もすべて管理画面で登録できる。
> このファイルのSQL手順は「複数日をまとめて一括登録したいとき」と
> 「TACOS Party等の特別枠（idが日付と異なる日程）を作るとき」に使う。

## 手順（3ステップ）

### 1. 下のテンプレートをコピーして、月のファイルを作る

`line-reservation/migrations/2026-08-sessions.sql` のような名前で保存する。
`【 】`の箇所をすべて埋めること。**埋めずに実行しないこと。**

```sql
-- 2026-08: 8月の日程追加
-- 値の並び: id, date, display_date, title, food, trainers, morning_run,
--           capacity, is_open, bento_json, has_tacos, note, time_label
INSERT OR IGNORE INTO sessions VALUES
  ('2026-08-02','2026-08-02','8/2（日）','【クラス名】',
   '【お弁当メニュー（なければNULL）】','【担当（例: GO, みどり（KITCHEN））】',【朝RUNあり=1/なし=0】,10,1,
   '【bento_json（下の説明参照。なければNULL）】',
   0,'【備考（例: 朝RUN 6:30〜あり）】',NULL),
  ('2026-08-09','2026-08-09','8/9（日）','【クラス名】',
   NULL,NULL,0,10,1,NULL,0,NULL,NULL),
  ('2026-08-16','2026-08-16','8/16（日）','【クラス名】',
   NULL,NULL,0,10,1,NULL,0,NULL,NULL),
  ('2026-08-23','2026-08-23','8/23（日）','【クラス名】',
   NULL,NULL,0,10,1,NULL,0,NULL,NULL),
  ('2026-08-30','2026-08-30','8/30（日）','【クラス名】',
   NULL,NULL,0,10,1,NULL,0,NULL,NULL);
```

### 2. 本番D1に流す（1回だけ）

```bash
wrangler d1 execute line-harness --file=line-reservation/migrations/2026-08-sessions.sql --remote
```

### 3. LIFFで表示確認

LINEアプリで予約フォームを開き、8月の日程が並ぶことを確認する。
**日程追加だけならWorkerのデプロイもKVの更新も不要**（DBを読むだけなので即反映）。

## 各列の意味

| 列 | 意味 |
|---|---|
| `id` | 通常の朝クラスは日付と同じにする（例: `2026-08-02`）。**TACOS Partyのような特別枠は必ず `2026-08-02-tacos` のように日付＋サフィックスにする**。idと日付が違うものは「特別枠」と判定され、回数券の対象外になり、確認・リマインドの文面も朝クラス用の定型文（AM7:30・動きやすい服装で）を使わなくなる |
| `date` | 開催日 `YYYY-MM-DD` |
| `display_date` | フォームに出す表示（例: `8/2（日）`） |
| `title` | クラス名 |
| `food` | お弁当・朝ごはんの紹介文。なければ `NULL` |
| `trainers` | 担当の表示。なければ `NULL` |
| `morning_run` | 朝RUNあり=1／なし=0。1にするとフォームに🏃タグが出て、参加確認の質問が表示される |
| `capacity` | 定員（通常10）。満席後はさらに追加枠3席まで自動で受け付ける |
| `is_open` | 受付中=1。**お休みの日は0**（フォームに出なくなる） |
| `bento_json` | お弁当の選択肢。例: `'[{"name":"カオマンガイ","price":1300}]'`。価格未定は `"price":null`。なければ `NULL` |
| `has_tacos` | 使っていない（旧機能）。常に0でよい |
| `note` | 日程カードに出る補足（例: `朝RUN 6:30〜あり`） |
| `time_label` | 開催時間の表示。**NULLなら「AM7:30〜10:00」扱い**。特別枠だけ `'12:00〜21:00'` のように入れる |

## よくある操作

- **開催を中止にする**: `UPDATE sessions SET is_open = 0 WHERE id = '2026-08-16';`
  （既に予約が入っている場合は、キャンセル連絡を手動で行うこと。自動通知はない）
- **定員を変える**: `UPDATE sessions SET capacity = 8 WHERE id = '2026-08-02';`
- **お弁当を後から追加**: `UPDATE sessions SET food = '…', bento_json = '[…]' WHERE id = '…';`

## 忘れずにセットで直すもの

- **LP（hacos-hmc-lp.html）の「7月の体験を予約する」等の月名**は手書きなので、
  月が替わったらLP側の文言も更新する（`python3 build_index.py` → QA → 承認 → 公開）。
