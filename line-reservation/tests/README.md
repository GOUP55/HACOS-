# 予約フォームの自動テスト

`reserve.html`（LINE予約フォーム）を、本番に触らずに手元で動かしてチェックする仕組み。
本物のWorker・LINEアプリの代わりに、モックサーバーとLIFFスタブを使う。

2026-07-06 の回数券対応の検証で作成（10項目）。
フォームを変更したら、デプロイ前にこのテストを回すこと。

## 実行方法

```bash
cd line-reservation/tests
npm install
npm test
```

- Chromium の場所が標準（`/opt/pw-browsers/chromium`）と違う環境では
  `CHROMIUM_PATH=/path/to/chromium npm test` で指定する。
- 成功すると `kaisuken-ui.png`（回数券選択状態のスクリーンショット）が出力される。

## テスト内容（10項目）

1. 参加区分に「回数券」の選択肢がある
2. 料金表記「¥2,000／回」がある
3. 最初は「まとめ選択」ボタンが隠れている
4. 「会員」を選んでもボタンは出ない
5. 「回数券」を選ぶとボタンが表示される
6. 注意書き（繰り越し不可・月途中は残り回数分）がある
7. ボタンで今月の空きセッションだけが一括選択される
8. 満席・来月分は選ばれない
9. ボタンが「✓ 選択しました」表示に変わり無効化される
10. 既存区分（ビジター）に切り替えるとボタンが消える

## 管理画面のテスト（admin-page.test.mjs）

`/api/admin/reservations`（スタッフ用の予約管理画面・認証必須）のHTML生成と、
管理画面ルートが認証素通しの非APIパスに配線されていないこと、キャンセル可視化・
KITCHEN向け集計行・個人行の弁当表示・体験の確定/不成立ボタン・開催日管理を検証する（39項目）。
`cancel-fallback.test.mjs` はmigration未適用DBでのフォールバックと取消系の競合ガードを
実SQLite（D1と同エンジン）で検証する（14項目）。
`sessions-admin.test.mjs` は開催日の登録・編集・削除のSQL挙動
（二重登録防止・締切の反映・予約ありの削除拒否・操作ログ）を検証する（11項目）。
表示内容・満席の赤表示・XSS対策（表示名のタグ無害化）・noindex を確認し、
`admin-ui.png` にプレビューのスクリーンショットを出力する。
`npm test` でフォームのテストと合わせて実行される。

## 注意

- これはUI（見た目と動き）のテスト。API・DB・通知は本物を使わないため、
  サーバー側の動作確認は従来どおり本番デプロイ後にLINEアプリ内のLIFFで行う。
- `node_modules/` と `kaisuken-ui.png` はコミットしない（.gitignore 済み）。
