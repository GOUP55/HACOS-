公開前の品質チェックを実行してください。

対象: $ARGUMENTS（指定がなければ index.html と premium.html）

進め方:
1. hacos-qa サブエージェントに対象ファイルをレビューさせる
2. チェック観点: リンク切れ / 表示崩れ（モバイル390px基準）/ タイポ / 料金・日付・定員の BUSINESS_RULES.md との整合 / 実績表現に「効果には個人差があります」が付いているか / ビルド整合（hacos-hmc-lp.html と index.html の一致）
3. 指摘を重大度「高・中・低」に分けたリストで報告する
4. このコマンドでは**修正はしない**。指摘リストを見た私が次の指示を出す

指摘ゼロならその旨を1行で報告してください。
