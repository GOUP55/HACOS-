公開（main反映)の準備をしてください。これは不可逆に近い操作なので、必ず段階を踏むこと。

対象: $ARGUMENTS

手順:
1. `git fetch origin main` して main との差分（`git log origin/main..HEAD --oneline` と `git diff origin/main --stat`）を確認する
2. TEAM_WORKFLOW.md の公開ゲート条件を確認する: QAの高/中指摘ゼロか・ダミー計測ID/プレースホルダ/下書きが混ざっていないか・ビルド整合が取れているか。未確認なら先に hacos-qa でチェックする
3. 「公開されるもの一覧」を非エンジニア向けの言葉で私に提示し、**承認を待つ**
4. 私が承認したら main への PR を作成し、URLを共有する。**mainへの直接pushは絶対にしない**
5. 承認が得られない・条件を満たさない場合は、何が足りないかを報告して止まる

注意: サンプル文言・仮の数値・テスト用コードが1つでも含まれるなら公開に進まない。
