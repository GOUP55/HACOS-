# 重量級タスクのAI投入プレイブック（モデル非依存）

> 2026-07-07作成。Fable 5利用期間の学びを「どのAIモデルでも再現できる形」にした正本。
> モデルの使い分け原則は `AI_COST_GUIDE.md`、チームの品質ループは `TEAM_WORKFLOW.md` を参照。
> このファイルは「重い仕事の**投げ方・測り方・課金判断**」だけを扱う。

---

## 1. 仕分け基準（A/B/C）— 新しい仕事が来たらまずここ

| 級 | 目安 | 投入先 |
|---|---|---|
| **A：最重量級** | 丸1日〜数日かかる／工程が多い／材料が多い／途中で判断が要る／ずっと後回しにしている | 最上位モデル（あれば）。無ければ `hacos-director` で分割してSonnetチームへ |
| **B：中量級** | 数時間かかる定型的な仕事 | Sonnetエージェント（frontend/copywriter/growth）。新モデルが来たら比較テストの題材にする |
| **C：軽量級** | 調べ物・短文修正・整形・チェック | Haiku（`hacos-ops`）または自分。上位モデルには絶対に投げない |
| **AI外** | 対人確認・管理画面操作・本番デプロイ・経営判断（価格/締切） | 人間。AIは選択肢と下書きまで |

**A級の見分け方（1つでも当てはまればA）**：
- 完成までに「読む→設計する→作る→見直す」の全工程が必要
- 参照すべき資料が3つ以上ある
- 途中で「どっちにするか」の判断が2回以上発生する

## 3. 依頼テンプレート（A級を丸ごと渡す型）

工程を指示しない。**「完成の定義」と「裁量の範囲」を渡す**。この3点で足りる。

```
■ 背景（1〜2行）
何のためにやるか。例：8月に第2期プレミアム（¥300,000・定員3名）を売り出すため。

■ 完成の定義（何がどんな形式で揃ったら終わりか）
例：
- 配信文面一式（LINE 4本、各300字以内、drafts/ に .md で保存）
- LP改修案（変更箇所と理由の一覧。実装はまだしない）
- 逆算カレンダー（締切から逆算した日付入り表）
- 全体をQA（hacos-qa）に通し、高/中指摘ゼロの状態

■ 裁量の範囲
勝手に進めてよい：構成・文面表現・ファイル分割・下位エージェントへの割り振り
確認が必要：価格・締切・実績数値に触れる変更、main反映、外部送信
正本：料金・締切は BUSINESS_RULES.md に従う。食い違ったらコードが正。
```

**渡すときの注意（実測からの学び）**：
- 出力形式を必ず指定する（字数・箇条書き・保存先）。出力トークンは入力の約5倍の単価
- 「判断に必要な情報が足りなければ先にまとめて質問して」を1行足す
- 完成の定義に「QA通過」を入れると品質ループが自動で回る

## 4. 消費量の測定シート（タスク1本ごとに記録）

AIからは使用量メーターが見えない。**投げる直前と完了直後に、自分で使用量画面を記録する**。

| 項目 | 記録 |
|---|---|
| タスク名 | |
| モデル | |
| 開始時の使用量%（スクショ推奨） | |
| 終了時の使用量% | |
| **このタスクの消費 = 差分%** | |
| かかった時間（自分の拘束時間） | |
| 成果物の再利用可否（手順書化した？） | |

**API価格での概算式**（実測が取れないときの目安。単価は変わるので都度確認）：
> 概算コスト ≒ 入力トークン数 × 入力単価 + 出力トークン数 × 出力単価
> 例：入力100万tk × $10/M + 出力5万tk × $50/M = $12.5（重量級1本の目安レンジは$10〜30）

**月間換算**：重量級 n本/月 × 1本あたり実測コスト ＋ 中量級 m本 × 実測コスト。この数字が月額サブスク費を超えるかで課金を判断する。

## 5. 比較テストの手順（新モデルが来るたびに再利用）

1. B級（中量級・定型）の実タスクを1〜2個選ぶ
2. **同じ指示文**を新旧モデルに投げ、出力を2つ保存する
3. 第三のAIセッションに以下の採点プロンプトを貼る：

```
同じ依頼に対する2つのAIの出力を貼ります。
正確性・網羅性・実務でそのまま使える度・指示の意図の汲み取りの4観点で、
それぞれ10点満点で採点し、差がついた箇所を具体的に指摘してください。
僅差なら僅差と正直に言ってください。
【依頼文】（貼る）
【出力1】（貼る）
【出力2】（貼る）
```

4. 判定：**大差**（合計5点差以上 or 実務使用度で3点差以上）→ そのタスク種は上位モデル候補。**僅差** → 通常モデルで確定

## 6. 課金の意思決定基準（3択）

| 選択 | 条件 |
|---|---|
| 課金継続 | 大差タスクが複数あり、月間換算コストがその価値（時給換算・売上寄与）に見合う |
| スポット利用 | 大差タスクは年数回の重い仕事だけ → 必要時のみ小額単位でクレジット購入 |
| 見送り | 僅差ばかり → 通常モデル＋このプレイブックで十分。堂々と見送る |

**原則：「最強モデルだから課金する」ではなく「自分の数字が課金を正当化するか」で決める。**

## 7. オーナーの好み・修正傾向メモ（依頼時にAIへ渡す）

- 結論を先に。読み手の次の行動が変わらない詳細は削る（ただし矢印連鎖・省略記号で圧縮しない）
- 数字は正本（BUSINESS_RULES.md）とコードで裏取りしてから書く。記憶で書かない
- 実績・効果の表現は盛らない。「効果には個人差があります」を必ず付ける
- 押し売りしないトーン（温かく誠実）。迷い・リスクのある文面ほど短く
- 質問は1回に1つ。曖昧でもまず解釈を出してから確認する
- 不可逆な操作（main反映・配信・削除）は必ず事前確認
